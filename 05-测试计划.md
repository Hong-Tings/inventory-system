# 进销存管理系统 — 测试计划

## 一、后端单元测试（JUnit 5 + Mockito）

| 服务 | 测试点 | 优先级 |
|------|--------|--------|
| **PurchaseOrderService** | ① submit：库存增加、加权平均成本价计算 ② cancel：库存回滚、均价重算 ③ void：状态校验 ④ 单号生成唯一性 | P0 |
| **SalesOrderService** | ① submit：FIFO扣减、库存不足拦截 ② cancel：库存回滚 ③ void：已出库不可作废 | P0 |
| **StockTakeService** | ① create：全盘自动加载商品 ② addItem：账面数量正确 ③ adjust：库存调整、仓库过滤 ④ 审核→调整状态流转 | P0 |
| **TransferService** | ① submit：源仓扣减、目标仓增加、成本价沿用 ② cancel：双向回滚 ③ 禁止同仓库调拨 | P0 |
| **WarehouseService** | ① enrichStats：商品总数、库存金额计算 ② delete：有库存时禁止删除 | P1 |
| **ProductService** | ① 商品作废前检查库存引用 ② 自动编码生成 | P1 |

## 二、API 测试（test-api.sh 增强）

| 场景 | 说明 |
|------|------|
| **正常流程** | 登录→新增商品→采购入库→销售出库→盘点→调拨 |
| **异常场景** | ① 密码错误 ② 库存不足出库 ③ 编码重复 ④ 必填字段缺失 ⑤ 无权访问 |
| **筛选查询** | 入库单/出库单/库存/盘点/调拨 各列表的分页、筛选、排序、搜索 |
| **导出** | 商品/供应商/客户/库存/入库/出库 Excel 导出 |
| **作废/取消** | 草稿取消→作废、已入库取消→作废、已出库取消→作废 |

## 三、业务场景测试（E2E 完整流程）

| 场景 | 步骤 |
|------|------|
| **日常业务** | 登录→新增商品(2个)→采购入库(批次1@10元)→采购入库(批次2@20元)→销售出库(部分)→查看库存金额→查看流水 |
| **异常处理** | 取消入库→验证库存回滚→作废单据→验证已作废列表 |
| **调拨** | 新建调拨单→提交→验证源仓扣减→验证目标仓增加 |
| **盘点** | 全盘→录入差异→审核→调整→验证库存更新 |
