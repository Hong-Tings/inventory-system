## 3. 数据库详细设计

### 3.1 核心表结构

#### 3.1.1 商品表 `product`

```sql
CREATE TABLE `product` (
  `id`          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
  `code`        VARCHAR(64)  NOT NULL COMMENT '商品编码',
  `name`        VARCHAR(200) NOT NULL COMMENT '商品名称',
  `category_id` BIGINT       NULL     COMMENT '分类ID',
  `spec`        VARCHAR(200) NULL     COMMENT '规格型号',
  `unit`        VARCHAR(20)  NOT NULL COMMENT '单位（个/箱/KG等）',
  `purchase_price` DECIMAL(10,2) NULL COMMENT '采购参考价',
  `sale_price`  DECIMAL(10,2) NULL COMMENT '销售参考价',
  `image_url`   VARCHAR(500) NULL     COMMENT '商品图片',
  `min_stock`   INT          DEFAULT 0 COMMENT '最低库存（安全库存）',
  `max_stock`   INT          DEFAULT 0 COMMENT '最高库存',
  `status`      TINYINT      DEFAULT 1 COMMENT '状态 0=停用 1=启用',
  `remark`      VARCHAR(500) NULL     COMMENT '备注',
  `deleted`     TINYINT      DEFAULT 0 COMMENT '逻辑删除',
  `create_time` DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_code` (`code`),
  KEY `idx_category` (`category_id`),
  KEY `idx_name` (`name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品表';
```

## 4. 后端接口设计

### 4.1 接口规范

- 基础路径：`/api/v1`
- 响应格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1714204800000
}
```

- 分页请求：`/api/v1/{module}/page?page=1&size=10&sort=create_time&order=desc`
- 分页响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [],
    "total": 100,
    "page": 1,
    "size": 10
  }
}
```

## 8. 关键业务流程

### 8.1 采购入库流程

```
[小程序/Web] 选择供应商 + 仓库 + 库位
       │
       ▼
[小程序/Web] 扫描/选择商品，录入数量、单价、批次
       │
       ▼
       保存 → 状态 = 草稿
       │
       ▼
       确认提交 → 状态 = 已入库
       │
       ├─ 校验：商品状态、库存上限
       ├─ 事务：批量插入明细
       ├─ 事务：更新 inventory.quantity += 入库数量
       └─ 事务：插入 inventory_log（PURCHASE_IN）
```

### 8.2 销售出库流程

```
[小程序/Web] 选择客户 + 仓库
       │
       ▼
[小程序/Web] 选择商品，输入数量（可查看各批次库存）
       │
       ▼
       保存 → 状态 = 草稿
       │
       ▼
       确认提交 → 状态 = 已出库
       │
       ├─ 校验：库存充足、商品状态
       ├─ 事务：更新 inventory.quantity -= 出库数量
       ├─ 事务：插入 inventory_log（SALES_OUT）
       └─ 规则：低于安全库存 → 告警
```

### 8.3 盘点流程

```
[Web] 创建盘点单（选择仓库/库位/全盘或抽盘）
      │
      ▼
[小程序/Web] 扫码盘点点位，录入实盘数量
      │
      ▼
      自动计算差异（账面 - 实盘）
      │
      ▼
[Web] 审核盘点单
      │
      ├─ 差异在允许范围 → 直接通过
      └─ 有差异 → 填写差异原因
      │
      ▼
[Web] 执行调整 → 生成调整单
      │
      ├─ 事务：更新 inventory.quantity
      └─ 事务：插入 inventory_log（STOCKTAKE）
```

## 9. 部署架构

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    volumes: [./data/mysql:/var/lib/mysql]
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: inventory

  server:
    build: ./inventory-server
    ports: ["8080:8080"]
    depends_on: [mysql]
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/inventory

  admin:
    build: ./inventory-admin
    ports: ["80:80"]
    depends_on: [server]

  nginx:
    image: nginx:alpine
    ports: ["443:443", "80:80"]
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on: [admin, server]
```
