# 2026-04-29 生产部署准备

## 1. 修复软删除后编码重复问题（DuplicateKeyException）

**问题**：Product/Warehouse/Customer/Supplier 四个基础资料实体有 `@TableLogic`，编码生成时的 `selectPage`/`selectCount` 自动过滤了已软删除记录（`deleted=1`），导致生成重复编码，插入时报 `DuplicateKeyException`。

**修复**：
- 每个 Mapper 加两个原始 SQL 方法，绕过 `@TableLogic`：
  - `countAllByCode(code)` — `SELECT COUNT(*) FROM table WHERE code = #{code}`
  - `selectMaxCodeByPrefix(prefix)` — `SELECT code FROM table WHERE code LIKE CONCAT(#{prefix}, '%') ORDER BY code DESC LIMIT 1`
- Service 的 `generateXxxCode()` 改用这两个方法计算序列号 + 防重复

**涉及文件**：
- `product/mapper/ProductMapper.java`
- `warehouse/mapper/WarehouseMapper.java`
- `customer/mapper/CustomerMapper.java`
- `supplier/mapper/SupplierMapper.java`
- `product/service/ProductService.java`
- `warehouse/service/WarehouseService.java`
- `customer/service/CustomerService.java`
- `supplier/service/SupplierService.java`

**验证**：采购入库/销售出库/调拨/盘点没有此问题（它们没有 `@TableLogic`，编码生成能看到所有记录）

## 2. 回收站关联名称显示

**问题**：已作废列表中，采购入库单的供应商/仓库名称、销售出库单的客户/仓库名称、调拨单的调出/调入仓库、盘点单的仓库名称为空。原因是这些字段都是 `@TableField(exist = false)`，回收站直接用 `mapper.selectList()` 查没有关联填充。

**修复**：
- 给 SupplierMapper/CustomerMapper/WarehouseMapper 加 `selectAnyById(id)`（原始SQL，忽略 `@TableLogic`）
- `RecycleBinController.list()` 中遍历已作废订单，关联查询并填充名称

**涉及文件**：
- `supplier/mapper/SupplierMapper.java`
- `customer/mapper/CustomerMapper.java`
- `warehouse/mapper/WarehouseMapper.java`
- `common/controller/RecycleBinController.java`

## 5. 部署方案

创建了 `inventory-server/DEPLOY.md`，包含：
- 后端 JAR 打包 + 生产配置 `application-prod.yml`
- 前端 Vite 构建 + Nginx 部署 + 反向代理
- Windows 简化方案
- 数据库备份脚本
- 检查清单

**生产与开发主要差异**：
| 项目 | 开发 | 生产 |
|------|------|------|
| 数据库账号 | root/123456 | 专用账号+随机密码 |
| SQL日志 | stdout（调试用） | NoLoggingImpl（关掉） |
| 上传路径 | 相对路径 | 固定绝对路径 |
| JWT密钥 | 写死 | 环境变量/文件注入 |
| CORS | Java配置 | Nginx同域访问 |
| 数据库备份 | 不需要 | 每日定时备份 |
