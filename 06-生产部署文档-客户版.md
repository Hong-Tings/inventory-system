# 进销存管理系统 — 生产部署说明

> 本文档面向客户，说明部署上线需要准备什么以及如何操作。

---

## 一、需要准备什么

| 项目 | 说明 | 参考费用 |
|------|------|----------|
| **云服务器** | 建议 4 核 8G 内存，5Mbps 带宽 | 约 500-1000 元/年 |
| **域名** | 1 个，用于电脑端访问后台（非必需） | 约 50 元/年 |
| **微信小程序账号** | 企业主体，用于发布小程序 | 约 300 元/年 |
| **操作系统** | Windows Server 或 Linux（推荐 Linux） | 免费 |

> 如果只在公司内部使用，不需要域名，直接用 IP 地址访问即可。

---

## 二、部署方式选择

### 方式一：Docker 部署（推荐）

**前提：** 服务器安装 Docker（技术人员可操作）

```
1. 将整个项目上传到服务器
2. 打开命令行，进入项目目录
3. 运行命令：docker-compose up -d
4. 等待几分钟，所有服务自动启动完成
```

完成后访问：
- 管理后台：`http://服务器IP`
- 小程序：需要在 request.js 中修改后端地址后编译上传

### 方式二：手动部署

**前提：** 服务器安装 JDK 17、MySQL 8.0、Node.js 20

```
1. 导入数据库脚本
2. 编译并启动后端
3. 编译并部署管理后台
4. 修改小程序后端地址并上传微信
```

## 三、需要技术人员配置的参数

以下三项需要技术人员根据实际环境修改：

### 3.1 数据库密码

在 `application.yml` 中修改 `spring.datasource.password`，默认 `123456`。

### 3.2 小程序后端地址

在 `inventory-miniapp/src/api/request.js` 中修改：

```js
const BASE_URL = 'http://你的服务器IP或域名:8080/api/v1'
```

### 3.3 文件上传路径

默认图片存在服务器上的 `uploads/` 目录，如需修改：

```yaml
app.upload-base-path: /自定义路径/uploads
```

## 六、日常维护

### 备份数据库

```bash
# 导出数据库（技术人员操作）
mysqldump -u root -p inventory > backup_20260430.sql
```

### 重启服务

如果服务器重启或系统异常：

```bash
# Docker 方式
docker-compose restart

# 手动方式
# 重新运行 java -jar xxx.jar 启动后端
```

### 图片备份

上传的商品图片在 `uploads/images/` 目录，迁移服务器时需要一并打包。

## 七、常见问题

### Q：网页打不开？
检查服务器是否开放了端口（80、8080），云服务器需要在安全组中放行。

### Q：小程序连不上服务器？
确保小程序中的 `BASE_URL` 填的是公网 IP 或域名，且服务器 8080 端口已开放。

### Q：上传图片不显示？
检查 `uploads/` 目录是否存在，是否有读写权限。
