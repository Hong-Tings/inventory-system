# 小程序功能修改以及添加优化记录

日期: 2026-05-13 ~ 2026-05-15

## 一、列表页优化

### 1. 采购入库单列表 ✅
- 添加状态筛选（草稿/待审批/已入库/已取消）
- 添加日期筛选（开始/结束日期选择器）
- 添加仓库级联筛选
- 待审批卡片右上角角标提示待审批数量
- 「筛选」按钮折叠/展开筛选栏

### 2. 销售出库单列表 ✅
- 同上（状态/日期/仓库筛选 + 待审批角标）

### 3. 库存盘点列表 ✅
- 搜索输入框增大（字号 15px，内边距 14px）
- 方式/状态筛选折叠到「筛选」按钮中
- 修复卡片点击无跳转问题（已添加 `@click="goDetail"`）

### 4. 库存调拨列表 ✅
- 添加状态筛选
- 添加日期筛选

### 5. 库存流水列表 ✅
- 两个搜索框合并为一个（同时搜索商品名称/编码和关联单号）
- 添加仓库级联筛选，折叠到「筛选」按钮中
- 后端搜索逻辑改为 OR 关系（商品匹配 OR 单号匹配）

### 6. 商品列表 ✅
- 名称搜索和编码搜索合并为一个输入框
- 排序改为面板式展开（库存/采购价/销售价，每项支持 ↓↑ 切换）
- 已有排序时点击按钮直接切换方向，无需打开面板
- 添加启用/停用状态筛选
- 重置按钮同时清除搜索、筛选、排序状态

## 二、新增功能

### 7. 待审批列表 ✅
- 新增页面 `pages/pending/list.vue`
- 支持按模块切换（采购/销售/调拨/盘点）
- 首页「单据列表」中增加待审批入口
- 点击卡片跳转到对应单据详情页

### 8. 保存草稿/提交审批分开 ✅
- 采购/销售/调拨新建页底部同时显示「保存草稿」和「确认入库/出库/调拨」两个按钮
- 「保存草稿」仅保存为草稿状态，不提交审批
- 保存成功后自动返回列表页
- 保存草稿前校验必填项（仓库+商品）

## 三、权限完善

### 9. 后端权限加固 ✅
- `stock-take/{id}/adjust` 添加 `@SaCheckRole("role_1")`（盘点调整仅管理员）
- 商品 CRUD（创建/更新/删除/恢复）全部限制为管理员
- 仓库/供应商/客户创建/更新限制为管理员

### 10. 小程序权限校验 ✅
- 盘点详情页「审核通过」「执行调整」按钮对员工隐藏
- 其余页面不展示任何管理员专属按钮

## 四、UI/UX 改进

### Tab 栏图标 ✅
- 首页、入库、出库、库存、我的 五个 Tab 添加图标
- 图标由用户提供，放置在 `static/tabbar/` 和 `src/static/tabbar/`

### 首页精简 ✅
- 去掉冗余的预警卡片（保留统计卡片区的预警数字）

### 筛选交互优化 ✅
- 日期改为原生日期选择器（`picker mode="date"`）
- 所有页面的重置按钮统一清除全部筛选条件

## 五、Bug 修复

### 后端 Bug
- 采购入库 `page()` 查询缺少 `status` 条件过滤（已修复）
- 库存流水搜索 `productName` 和 `refOrderNo` 使用 AND 关系导致搜索无效（改为 OR）
- `warehouseCode` 赋值缺少花括号导致可能空指针（已修复）

### 前端 Bug
- 采购入库状态筛选无效（后端缺失 + 前端 ref 赋值方式修正）
- 商品列表排序状态重置未清除
- 保存草稿未校验必填项导致数据库报错
- 待审批列表 `onShow` 导入来源错误（应来自 `@dcloudio/uni-app`）
