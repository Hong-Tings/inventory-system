# 安全与体验优化记录

> 修改日期: 2026-05-02
> 涉及端: 后端 (inventory-server) + 管理后台 (inventory-admin) + 小程序 (inventory-miniapp)

---

## 一、后端安全修复

### 1. 全局认证拦截器
- **文件**: `common/config/WebMvcConfig.java`
- **内容**: 新增 Sa-Token SaInterceptor 全局拦截器，覆盖 `/api/**` 和 `/uploads/**`，排除登录接口 `/api/v1/auth/login`
- **原因**: 之前无全局认证，任何接口都可以不带 token 直接访问

### 2. 文件上传扩展名校验修复
- **文件**: `common/controller/FileController.java`
- **内容**: 上传文件扩展名校验从 `contains` 改为精确 `Set.of()` 匹配
- **原因**: 原逻辑用 `contains` 判断，`.jpgx` 等非法扩展名也能通过校验

### 3. 负库存防护
- **文件**: `purchase/service/PurchaseOrderService.java` (cancel 方法)、`transfer/service/TransferService.java` (cancel 方法)
- **内容**: 取消入库/调拨时增加库存回退后的负数校验，库存不足时抛出业务异常
- **原因**: 取消操作直接扣减库存，未校验扣减后是否为负数

### 4. 禁止同账号并发登录
- **文件**: `auth/AuthController.java`
- **内容**: 登录时设置 `SaLoginConfig.setIsConcurrent(false)`，同一账号新登录自动踢掉旧会话
- **配置**: `application.yml` 中 `is-concurrent` 同步改为 `false`
- **原因**: 之前同一账号可在多处同时登录，存在安全隐患

### 5. 记住我功能（Token 延长有效期）
- **文件**: `auth/AuthController.java`
- **内容**: 登录接口新增 `rememberMe` 参数，为 `true` 时 token 有效期延长至 30 天；否则保持默认（24 小时，30 分钟不活跃过期）
- **原因**: 小程序用户和勾选"记住密码"的管理员需要长期保持登录

### 6. NotRoleException 全局异常处理
- **文件**: `common/exception/GlobalExceptionHandler.java`
- **内容**: 新增 `NotRoleException` 专用处理器，返回 `{ code: 403, message: "权限不足，无法执行此操作" }`
- **原因**: 之前角色校验失败没有专用处理器，走到通用 Exception 兜底，直接返回原始异常信息 `NotRoleException: 无此角色：role_1`

### 7. 系统日志接口权限放开
- **文件**: `system/controller/SystemLogController.java`
- **内容**: 移除类级别 `@SaCheckRole("role_1")` 注解，所有已登录用户均可查看操作日志
- **原因**: Dashboard 加载时请求 `/system/log/page`，员工账号登录即触发 403 报错

### 8. 库存盘点实体映射修复
- **文件**: `stocktake/entity/StockTake.java`
- **内容**: `locationId` 字段添加 `@TableField(exist = false)`
- **原因**: stock_take 表无 location_id 列，MyBatis-Plus 自动映射时报 `Unknown column 'location_id'`

### 9. 盘点调整跨仓库问题修复
- **文件**: `stocktake/service/StockTakeService.java`
- **内容**: adjust() 和 addItem() 方法查询库存时补充 warehouseId 过滤条件
- **原因**: 缺少仓库条件导致跨仓库盘点时错误清零其他仓库的同商品库存

### 10. 移动加权平均成本价
- **文件**: `inventory/entity/Inventory.java`、`purchase/service/PurchaseOrderService.java`、`transfer/service/TransferService.java`、`warehouse/service/WarehouseService.java`
- **内容**: inventory 表新增 cost_price 字段，每次采购入库按（原库存总金额+本次进货金额）÷（原库存总数量+本次进货数量）重算加权平均成本价
- **原因**: 原方案使用商品采购参考价×库存数量，进价波动时金额不准确

## 二、管理后台前端优化

### 1. 危险操作确认弹窗（8 处）
- **文件**: PurchaseDetail.vue、SalesDetail.vue、TransferDetail.vue、StockTakeDetail.vue、PurchaseForm.vue、SalesForm.vue、TransferForm.vue、ProductList.vue
- **内容**: 取消入库/出库、确认调拨、取消调拨、审核通过、执行调整、确认入库/出库/调拨、启用/停用商品等操作前弹出 ElMessageBox.confirm 确认框
- **原因**: 原逻辑点击即执行，无二次确认，误操作风险高

### 2. 表单逐行校验
- **文件**: PurchaseForm.vue、SalesForm.vue、TransferForm.vue
- **内容**: 提交前校验每行明细的商品选择、数量（>0）、单价/售价是否有效，不通过时提示具体行号
- **原因**: 原逻辑只校验仓库和明细行数，行内数据可以为空或非法

### 3. 分页 size-change 重置页码（11 个列表页）
- **文件**: PurchaseList、SalesList、TransferList、StockTakeList、ProductList、CustomerList、SupplierList、WarehouseList、InventoryList、InventoryLog、OperationLog、UserList
- **内容**: `@size-change` 事件改为 `query.page = 1; fetchData()`，切换每页条数时自动回到第 1 页
- **原因**: 原逻辑切 每页条数后不重置页码，可能请求到超出范围的页导致空数据

### 4. 表单未保存离开提醒（3 个表单页）
- **文件**: PurchaseForm.vue、SalesForm.vue、TransferForm.vue
- **内容**: 添加 `onBeforeRouteLeave` 路由守卫，表单有修改未保存时弹确认框
- **原因**: 原逻辑误触返回直接丢失已填写数据

### 5. 详情页错误处理（4 个详情页）
- **文件**: PurchaseDetail.vue、SalesDetail.vue、TransferDetail.vue、StockTakeDetail.vue
- **内容**: `fetchDetail` 方法添加 catch 块，请求失败时显示错误提示
- **原因**: 原逻辑只有 finally，请求失败时页面空白无反馈

### 6. Dashboard 加载状态
- **文件**: Dashboard.vue
- **内容**: 新增 `loading` ref，页面加载时显示 v-loading 遮罩
- **原因**: 数据加载期间页面显示空数据，体验差

### 7. 导出 Excel 认证修复（10 个列表页）
- **文件**: 所有含导出功能的列表页
- **内容**: 导出从 `window.open(url)` 改为 `downloadFile(url, filename)`（axios Blob 下载，带 Authorization 请求头）
- **原因**: 添加全局认证拦截器后，window.open 不带 token 会被 401 拦截

### 8. 401 拦截器修复
- **文件**: `api/request.ts`
- **内容**: 401 处理后添加 `return Promise.reject(error)` 阻断后续处理，避免重复弹出错误提示

### 9. 403 权限不足拦截
- **文件**: `api/request.ts`
- **内容**: 新增 HTTP 403 状态码拦截，显示"权限不足，无法执行此操作"
- **原因**: 原逻辑无 403 处理，员工访问管理员接口时显示原始异常信息

### 10. 登录页记住密码
- **文件**: `views/login/Login.vue`、`store/user.ts`
- **内容**: 登录页新增"记住密码"复选框，勾选后账号密码保存到 localStorage 自动填充，登录请求传递 `rememberMe` 参数
- **原因**: 用户每次需手动输入账号密码，体验差

## 三、小程序优化

### 1. 登录默认保持登录
- **文件**: `store/user.js`
- **内容**: 登录请求默认传递 `rememberMe: true`，token 有效期 30 天
- **原因**: 小程序用户期望登录后长期保持，频繁重新登录体验差

### 2. 403 权限不足处理
- **文件**: `api/request.js`
- **内容**: 请求拦截器新增业务码 403 处理，显示"权限不足"
- **原因**: 原逻辑无 403 处理，员工访问管理员接口时显示原始错误
