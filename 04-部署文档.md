## 二、什么是 Docker？（通俗解释）

**Docker 就是一个"一键打包"工具。**

传统部署方式：
```
你买了一台新服务器 → 安装 JDK → 安装 MySQL → 安装 Nginx → 
配置环境变量 → 一个一个启动服务 → 出错了还要排查
```

Docker 方式：
```
写好一个配置文件 → 一条命令 → 所有服务自动装好启动
```

**好处：** 换服务器不用重新配环境，一条命令全部搞定。

**本项目实际上没有打包 Docker**，目前运行方式是手动启动各个服务。如果你想用 Docker 部署，需要额外做打包工作（后续章节会说明）。

### 7.2 管理后台配置

路径：`inventory-admin/vite.config.ts`

```typescript
server: {
  port: 3000,                    // 前台端口
  proxy: {

      target: 'http://localhost:8080',  // 代理到后端
    },
  },
},
```

## 八、生产环境部署

### 8.1 方法一：手动部署（推荐你使用这种方式）

#### 第一步：后端打包运行

```bash
# 编译打包
cd inventory-server
mvn package -DskipTests -q

# 运行（nohup 让它在后台持续运行）
nohup java -jar target/inventory-server.jar > server.log 2>&1 &
```

#### 第二步：管理后台打包

```bash
cd inventory-admin
npm run build
```

产物在 `dist/` 目录，将其部署到 Nginx 或直接打开 `dist/index.html`。

Nginx 配置示例：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /path/to/inventory-admin/dist;
        index index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
    }
}
```

#### 第三步：小程序发布

```bash
cd inventory-miniapp
# 先修改 request.js 中的 BASE_URL 为生产地址
npm run build:mp-weixin
```

用微信开发者工具打开 `dist/build/mp-weixin/`，上传审核发布。

### 8.2 方法二：Docker 部署

> Docker 部署 = 不需要手动安装 JDK/MySQL/Node，一条命令启动全部。

#### 第一步：安装 Docker

**Windows：** 下载安装 Docker Desktop（https://www.docker.com/products/docker-desktop/）
**Linux：** `curl -fsSL https://get.docker.com | sh`

#### 第二步：一键启动

项目根目录已有 `docker-compose.yml`、`Dockerfile` 等文件，直接运行：

```bash
cd C:\Users\28135\Desktop\管理
docker-compose up -d
```

等待几分钟（第一次需要下载镜像和编译代码），然后访问：

| 服务 | 地址 |
|------|------|
| 管理后台 | http://localhost |
| 后端 API | http://localhost:8080 |
| Swagger 文档 | http://localhost:8080/doc.html |

#### 第三步：查看运行状态

```bash
docker-compose ps           # 查看三个容器是否都在运行
docker-compose logs -f      # 查看实时日志（Ctrl+C 退出）
```

#### 第四步：停止和重启

```bash
docker-compose stop          # 停止所有服务
docker-compose start         # 再次启动
docker-compose down          # 停止并删除容器
```

#### 文件和端口说明

| 文件 | 作用 |
|------|------|
| `docker-compose.yml` | 编排三个服务：mysql、server、admin |
| `inventory-server/Dockerfile` | 后端打包脚本（编译 + 运行） |
| `inventory-admin/Dockerfile` | 后台打包脚本（构建 + Nginx 部署） |
| `inventory-admin/nginx.conf` | Nginx 配置（代理 API 到后端） |

| 端口 | 作用 |
|------|------|
| `80` | 管理后台（浏览器直接访问） |
| `8080` | 后端 API |
| `3307` | MySQL（外部工具连接用，容器内部用 3306） |

> ⚠️ Docker 部署后，小程序 `request.js` 中的 `BASE_URL` 要改成你服务器的公网 IP 或域名。

## 九、常见问题

### Q1：后端启动报数据库连接失败
检查 MySQL 是否已启动，确认 `application.yml` 中的用户名密码正确。

### Q2：小程序打开空白
- 检查 `request.js` 中的 `BASE_URL` 是否正确
- 如果后端是本地，需用 `ipconfig` 查看本机 IP，不能用 `localhost`
- 手机和电脑必须在同一局域网

### Q3：后端启动后前端报跨域错误
`vite.config.ts` 已经配置了 proxy 代理，开发阶段直接访问 `localhost:3000` 即可，不会跨域。

### Q4：如何停止后端服务

```bash
# 查看后端的进程ID
ps aux | grep java
# 或 Windows 下
netstat -ano | findstr :8080

# 杀掉进程
kill -9 进程ID
```

### Q5：数据库密码怎么改
修改 `inventory-server/src/main/resources/application.yml` 中的 `spring.datasource.password`，然后重新编译后端：

```bash
cd inventory-server
mvn package -DskipTests -q
java -jar target/inventory-server.jar
```
