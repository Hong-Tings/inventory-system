# 进销存管理系统 — 项目方案书

> 版本：v1.0 | 日期：2026-04-27

---

## 一、项目概述

为植物/花卉类销售企业定制开发一套进销存管理系统，实现**采购入库、销售出库、库存盘点、库存查询、商品管理**全流程数字化，解决手工记账数据不准、追溯困难的问题。

## 二、系统构成

| 终端 | 说明 | 面向用户 |
|------|------|----------|
| **微信小程序** | 手机直接打开使用，扫码录入、出入库操作、盘点、查库存 | 仓库员、销售员 |
| **PC 管理后台** | 浏览器打开，数据管理、报表、系统配置 | 管理员、老板 |

## 三、核心功能

| 功能 | 描述 |
|------|------|
| **采购入库** | 选供应商→选仓库库位→添加商品（数量/批次）→确认，库存自动增加 |
| **销售出库** | 选客户→选仓库→添加商品→确认，库存自动扣减，低于安全库存预警 |
| **库存盘点** | 创建盘点单（全盘/抽盘）→录入实盘数量→自动计算差异→审核→盈亏调整 |
| **库存查询** | 按仓库/商品/库位多维度查库存，支持查看流水追溯每一笔变动 |
| **商品管理** | 商品信息维护（名称、编码、规格、价格、安全库存阈值） |
| **基础数据** | 供应商、客户、仓库、库位管理 |
| **统计报表** | 入库/出库汇总报表、库存预警列表 |
| **权限管理** | 管理员、仓库员、销售员角色分离 |

## 四、技术方案

全部采用**成熟稳定**的技术栈，不引入任何实验性框架：

| 层级 | 技术 |
|------|------|
| 后端 | Java 17 + Spring Boot 3.2 + MySQL 8.0 |
| PC 后台 | Vue 3 + Element Plus（浏览器直接访问） |
| 小程序 | UniApp（Vue 3 组合式 API，一套代码多端适配） |
| 部署 | Docker 容器化，支持云服务器一键部署 |

**技术特点：**
- Spring Boot + MyBatis-Plus 是 Java 领域最成熟的后端组合，社区庞大、问题易查
- 小程序采用 UniApp 框架，Vue 3 组合式 API，与 PC 后台技术栈统一，降低学习成本
- Sa-Token JWT 模式，无状态认证，无需额外部署 Redis
- MySQL 8.0 单库即可满足中小规模门店的数据存储需求
- Docker 部署，迁移、扩容方便

## 五、开发流程

```
第1-2周   需求确认 + UI 设计稿定稿
第3-5周   后端接口开发
第5-7周   小程序端开发（入库、出库、盘点、查库存）
第7-8周   PC 管理后台开发（商品管理、报表、系统管理）
第9周     前后端联调测试
第10周    验收交付 + 部署上线
```

## 六、交付物

|--------|------|

## 七、部署配置

| 资源 | 规格 |
|------|------|
| 云服务器 | 2 核 4G 起（建议 4 核 8G） |
| 带宽 | 5Mbps 起 |
| 域名 | 1 个（PC 后台用），需 ICP 备案 |
